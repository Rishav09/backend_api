# FastAPI Deployment Notes — AWS EC2 + PostgreSQL + Nginx

## 1. EC2 basics
- Instance: Ubuntu on EC2
- SSH uses key pair, not password
- SSH command:
  ```bash
  ssh -i key.pem ubuntu@PUBLIC_IP
  ```

## 2. Important ports
- `22` → SSH
- `8000` → FastAPI/Gunicorn app
- `80` → HTTP via Nginx
- `443` → HTTPS via Nginx + Certbot
- `5432` → PostgreSQL

## 3. Security group rules
Recommended inbound rules:
- SSH `22` → **My IP**
- HTTP `80` → **0.0.0.0/0**
- HTTPS `443` → **0.0.0.0/0**
- App `8000` → **My IP only** if testing directly
- PostgreSQL `5432` → **My IP only** if using pgAdmin remotely

## 4. PostgreSQL notes
Switch to postgres OS user:
```bash
sudo -i -u postgres
```

Important files:
- `/etc/postgresql/16/main/pg_hba.conf` → who can connect + auth method
- `/etc/postgresql/16/main/postgresql.conf` → server settings like `listen_addresses`

Check PostgreSQL listening:
```bash
sudo ss -ltnp | grep 5432
```

## 5. pgAdmin remote connection
Use:
- **Host** = EC2 **public IP**
- not private `172.31.x.x`

If pgAdmin times out:
- check security group rule for `5432`
- check PostgreSQL is listening on `0.0.0.0:5432`
- check `pg_hba.conf`

## 6. Use uv on the server
Because project already uses:
- `pyproject.toml`
- `uv.lock`

Install dependencies:
```bash
uv sync
```

Run app:
```bash
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## 7. Alembic
Run migrations:
```bash
uv run alembic upgrade head
```

If migration fails, check for:
- duplicate columns
- bad autogenerated migration code
- wrong migration ordering

## 8. Git workflow
`git pull` = fetch + merge latest code from GitHub

Normal deploy flow:
```bash
git pull origin main
uv sync
sudo systemctl restart fastapi
```

If schema changed:
```bash
uv run alembic upgrade head
sudo systemctl restart fastapi
```

## 9. systemd service
Purpose:
- run app in background
- survive SSH disconnect
- restart on crash
- start on reboot

Useful commands:
```bash
sudo systemctl daemon-reload
sudo systemctl start fastapi
sudo systemctl restart fastapi
sudo systemctl status fastapi
sudo systemctl enable fastapi
sudo journalctl -u fastapi -n 50 --no-pager
```

```bash
/home/rishav/app/src/.venv/bin/gunicorn -w 2 -k uvicorn_worker.UvicornWorker app.main:app --bind 0.0.0.0:8000
```

## 11. Nginx role
Architecture:
```text
Browser -> Nginx :80/:443 -> Gunicorn/FastAPI :8000
```

Nginx proxy config:
```nginx
server {
    listen 80;
    server_name rishavsapahia.xyz www.rishavsapahia.xyz 3.217.133.65;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }
}
```

Check and reload:
```bash
sudo nginx -t
sudo systemctl reload nginx
sudo systemctl restart nginx
sudo systemctl status nginx
```

Check listening ports:
```bash
sudo ss -ltnp | grep ':80'
sudo ss -ltnp | grep ':8000'
```

## 12. Domain mapping
Domain: `rishavsapahia.xyz`

Use **Elastic IP** on EC2.

GoDaddy DNS records:
- `A` record: `@` → Elastic IP
- `CNAME`: `www` → `rishavsapahia.xyz`

Check DNS:
```bash
dig rishavsapahia.xyz +short
dig www.rishavsapahia.xyz +short
```

## 13. Certbot HTTPS
Install and configure:
```bash
sudo snap install core
sudo snap refresh core
sudo snap install --classic certbot
sudo ln -s /snap/bin/certbot /usr/local/bin/certbot
sudo certbot --nginx -d rishavsapahia.xyz -d www.rishavsapahia.xyz
```

Test renewal:
```bash
sudo certbot renew --dry-run
```

## 14. Quick debugging checklist

### App not opening on browser
Check:
```bash
sudo ss -ltnp | grep 8000
nc -vz PUBLIC_IP 8000
```

### Domain not opening
Check:
```bash
dig rishavsapahia.xyz +short
nc -vz PUBLIC_IP 80
curl -I http://rishavsapahia.xyz
```

### FastAPI service failing
Check:
```bash
sudo systemctl status fastapi
sudo journalctl -u fastapi -n 50 --no-pager
```

### Nginx failing
Check:
```bash
sudo nginx -t
sudo systemctl status nginx
```

### PostgreSQL remote access failing
Check:
```bash
sudo ss -ltnp | grep 5432
sudo ufw status
nc -vz PUBLIC_IP 5432
```

## 15. Final architecture
```text
Domain -> Elastic IP -> EC2
                |
             Nginx :80/:443
                |
        Gunicorn + Uvicorn worker :8000
                |
           FastAPI backend
                |
           PostgreSQL on EC2
```

## 16. Safe habits
- Use public IP/domain from laptop, not private EC2 IP
- Keep SSH restricted to **My IP**
- Keep PostgreSQL private unless really needed
- Keep app port `8000` non-public after Nginx works
- Use Elastic IP for stable DNS
- Keep local and server dependency tooling consistent with `uv`
